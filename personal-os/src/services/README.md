### AI API via local Ollama proxy (Gemma light)

The app now routes AI calls through a local backend proxy (`server/index.js`) to Ollama.

Set in `.env`:

- `VITE_AI_PROXY_URL` — frontend target for proxy (default `http://localhost:8787`)
- `AI_PROXY_PORT` — local proxy port (default `8787`)
- `OLLAMA_BASE_URL` — Ollama API base URL (default `http://127.0.0.1:11434`)
- `OLLAMA_MODEL` — model tag (default `gemma2:2b`)
- `OLLAMA_TIMEOUT_MS` — request timeout to avoid hanging calls (default `45000`)
- `OLLAMA_NUM_CTX` — context window for CPU efficiency (default `2048`)
- `OLLAMA_NUM_PREDICT_DEFAULT` — default max generated tokens (default `512`)
- `VITE_LIFEMANAGER_FAST_MODE` — caps specialist output for faster CPU responses (default `true`)

Install + run Ollama:

```bash
ollama serve
ollama pull gemma2:2b
```

Run app + proxy together:

```bash
npm run dev:all
```

Troubleshooting:

- **Ollama unreachable**: confirm `ollama serve` is running and `OLLAMA_BASE_URL` is correct.
- **Model not found**: run `ollama pull gemma2:2b` (or your configured model tag).
- **AI proxy down**: ensure `npm run dev:server` starts without port conflicts.
- **Too slow on CPU**: keep `VITE_LIFEMANAGER_FAST_MODE=true`, lower `OLLAMA_NUM_PREDICT_DEFAULT` to `256`, and use shorter prompts.
