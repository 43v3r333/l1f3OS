# DEVELOPMENT RULES

## Execution Rules
- Always build in phases
- Stop after each phase
- Provide test instructions

## Complexity Rules
- Do not overengineer
- Avoid unnecessary abstractions
- Keep logic simple

## UI Rules
- Must use component library
- Must be mobile-first
- Must look modern and premium

## Code Rules
- Clean, readable code
- No large files
- Use reusable components

## Behavior Rules
- Do not jump ahead
- Do not add features not requested
- Always confirm before next phase
